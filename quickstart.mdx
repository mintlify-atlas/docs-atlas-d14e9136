## Prerequisites

Before you begin, ensure you have:
- A valid email address for account registration
- An API client (cURL, Postman, or your preferred HTTP client)
- Basic knowledge of REST APIs

<Steps>
  <Step title="Create Your Account">
    Navigate to the SentimentAPI registration page and sign up with your email and password.
    
    <Note>
      The platform uses the field name `contrasenia` (Spanish for password) in authentication requests.
    </Note>
  </Step>

  <Step title="Login and Get Your Token">
    After creating your account, authenticate to receive your access token. The token is valid for 2 hours.

    <CodeGroup>
    ```bash cURL
    curl -X POST https://api.sentimentapi.com/login \
      -H "Content-Type: application/json" \
      -d '{
        "email": "your@email.com",
        "contrasenia": "your_password"
      }'
    ```

    ```javascript JavaScript/Node.js
    const response = await fetch('https://api.sentimentapi.com/login', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        email: 'your@email.com',
        contrasenia: 'your_password'
      })
    });

    const data = await response.json();
    const token = data.token;
    ```

    ```python Python
    import requests

    response = requests.post(
        'https://api.sentimentapi.com/login',
        json={
            'email': 'your@email.com',
            'contrasenia': 'your_password'
        }
    )

    data = response.json()
    token = data['token']
    ```
    </CodeGroup>

    **Response:**
    ```json
    {
      "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "idUser": 123,
      "name": "John Doe",
      "role": "user"
    }
    ```

    <Warning>
      Store your token securely. It expires after 2 hours and should never be exposed in client-side code or version control.
    </Warning>
  </Step>

  <Step title="Make Your First Sentiment Analysis">
    Now that you have your token, you can analyze text sentiment using the `/sentiment` endpoint.

    <CodeGroup>
    ```bash cURL
    curl -X POST https://api.sentimentapi.com/sentiment \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer YOUR_TOKEN_HERE" \
      -d '[{
        "texto": "This product is amazing! I love it."
      }]'
    ```

    ```javascript JavaScript/Node.js
    const analyzeText = async (token, text) => {
      const response = await fetch(
        `${process.env.NEXT_PUBLIC_API_URL}/sentiment`,
        {
          method: 'POST',
          headers: {
            'Content-Type': 'application/json',
            'Authorization': `Bearer ${token}`
          },
          body: JSON.stringify([{ texto: text }])
        }
      );

      const results = await response.json();
      return results;
    };

    // Usage
    const results = await analyzeText(
      token,
      'This product is amazing! I love it.'
    );
    ```

    ```python Python
    import requests

    def analyze_sentiment(token, text):
        response = requests.post(
            'https://api.sentimentapi.com/sentiment',
            headers={
                'Content-Type': 'application/json',
                'Authorization': f'Bearer {token}'
            },
            json=[{'texto': text}]
        )
        return response.json()

    # Usage
    results = analyze_sentiment(
        token,
        'This product is amazing! I love it.'
    )
    ```
    </CodeGroup>

    **Request Format:**
    ```json
    [
      { "texto": "Your text to analyze" }
    ]
    ```

    <Note>
      The API accepts an array of text objects, allowing you to analyze multiple texts in a single request.
    </Note>
  </Step>

  <Step title="Understand the Response">
    The API returns an array of sentiment analysis results with predictions and confidence scores.

    **Response Format:**
    ```json
    [
      {
        "texto": "This product is amazing! I love it.",
        "prevision": "POSITIVO",
        "probabilidad": 0.956
      }
    ]
    ```

    **Response Fields:**
    - `texto` (string): The original text that was analyzed
    - `prevision` (string): The sentiment prediction - either `"POSITIVO"` or `"NEGATIVO"`
    - `probabilidad` (number): Confidence score between 0 and 1 (multiply by 100 for percentage)

    **Example with multiple texts:**
    ```json
    [
      {
        "texto": "This product is amazing!",
        "prevision": "POSITIVO",
        "probabilidad": 0.956
      },
      {
        "texto": "Terrible experience, would not recommend.",
        "prevision": "NEGATIVO",
        "probabilidad": 0.892
      }
    ]
    ```
  </Step>
</Steps>

## Common Issues

<AccordionGroup>
  <Accordion title="Authentication failed error">
    If you receive `"Credenciales incorrectas. Intenta de nuevo."`, verify that:
    - Your email and password are correct
    - You're using the field name `contrasenia` (not `password`)
    - The API URL is correct
  </Accordion>

  <Accordion title="Token not received">
    If you get `"No se recibió un token válido."`, ensure:
    - Your account is properly activated
    - The login response includes a valid token
    - You're parsing the response correctly
  </Accordion>

  <Accordion title="Service unavailable error">
    If you see `"El servicio de autenticación no está disponible."`, this indicates:
    - The authentication service is temporarily down
    - Network connectivity issues
    - Try again in a few moments or contact support
  </Accordion>
</AccordionGroup>
