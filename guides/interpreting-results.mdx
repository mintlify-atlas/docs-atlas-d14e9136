## Overview

SentimentAPI returns structured results for each analyzed text, including a sentiment prediction and confidence score. This guide helps you understand these values and make data-driven decisions.

## Result Structure

Every analysis returns a result object with three key fields:

```typescript
interface AnalisisResultado {
  texto: string;           // Original text analyzed
  prevision: 'POSITIVO' | 'NEGATIVO';  // Sentiment prediction
  probabilidad: number;    // Confidence score (0-1 range)
}
```

<Note>
This interface is defined in `app/sentiment/upload-csv/page.tsx:20-24` and used consistently across single and bulk analysis.
</Note>

## Understanding Prevision Values

The `prevision` field indicates the predicted sentiment category:

### POSITIVO

Indicates the text expresses positive sentiment, satisfaction, or approval.

**Examples:**
- "Excelente servicio, muy recomendado"
- "El producto superó mis expectativas"
- "Atención al cliente muy amable y eficiente"

<Tip>
Positive predictions are displayed with a green badge and smile icon in the UI (`app/sentiment/upload-csv/page.tsx:299-306`).
</Tip>

### NEGATIVO

Indicates the text expresses negative sentiment, dissatisfaction, or criticism.

**Examples:**
- "Terrible experiencia, nunca volveré"
- "Producto defectuoso, solicité reembolso"
- "La entrega tardó demasiado tiempo"

<Warning>
Negative predictions appear with a red badge and frown icon, signaling areas requiring immediate attention.
</Warning>

## Understanding Probabilidad (Confidence Score)

The `probabilidad` field represents the model's confidence in its prediction:

### Value Range

- **Type:** Float number between 0 and 1
- **Display:** Converted to percentage (0-100%)
- **Calculation:** `probabilidad * 100`

```typescript
// Display formatting (app/sentiment/page.tsx:154)
{(res.probabilidad * 100).toFixed(1)}%
```

### Confidence Levels

<AccordionGroup>
  <Accordion title="High Confidence: 80-100%">
    The model is very confident in its prediction. These results are highly reliable.
    
    **Example:**
    ```json
    {
      "texto": "Producto excelente, superó todas mis expectativas",
      "prevision": "POSITIVO",
      "probabilidad": 0.95
    }
    ```
    
    **Interpretation:** 95% confidence → Very clear positive sentiment
    
    **Action:** Trust this prediction for automated workflows and decision-making.
  </Accordion>

  <Accordion title="Medium Confidence: 60-79%">

    **Example:**
    ```json

      "texto": "El producto es bueno pero el precio es alto",
      "prevision": "POSITIVO",
      "probabilidad": 0.67
    }
    ```
    
    **Interpretation:** 67% confidence → Mixed sentiment with slight positive lean
    
    **Action:** Consider manual review for critical decisions.
  </Accordion>

  <Accordion title="Low Confidence: 50-59%">
    The model has low confidence. The text may contain ambiguous or neutral sentiment.
    
    **Example:**
    ```json
    {
      "texto": "El servicio es normal, nada especial",
      "prevision": "NEGATIVO",
      "probabilidad": 0.52
    }
    ```
    
    **Interpretation:** 52% confidence → Uncertain, possibly neutral sentiment
    
    **Action:** Manual review recommended before taking action.
  </Accordion>

## Decision Thresholds for Action

Use these guidelines to determine appropriate actions based on results:

### For Customer Feedback Analysis

<CodeGroup>
```javascript High Priority (Negative, High Confidence)
if (prevision === 'NEGATIVO' && probabilidad >= 0.80) {
  // Immediate action required
  priorityLevel = 'HIGH';
  action = 'Contact customer within 24 hours';
}
```

```javascript Medium Priority (Negative, Medium Confidence)
if (prevision === 'NEGATIVO' && probabilidad >= 0.60 && probabilidad < 0.80) {
  // Review and respond
  priorityLevel = 'MEDIUM';
  action = 'Add to review queue';
}
```

```javascript Positive Feedback (Positive, High Confidence)
if (prevision === 'POSITIVO' && probabilidad >= 0.80) {
  // Celebrate and promote
  priorityLevel = 'LOW';
  action = 'Consider for testimonials or case studies';
}
```

```javascript Uncertain Cases (Low Confidence)
if (probabilidad < 0.60) {
  // Manual review needed
  priorityLevel = 'REVIEW';
  action = 'Flag for human analysis';
}
```
</CodeGroup>

### For Content Moderation

```typescript
function determineModerationType(result: AnalisisResultado) {
  if (result.prevision === 'NEGATIVO' && result.probabilidad > 0.85) {
    return 'AUTO_FLAG'; // Automatic flagging
  } else if (result.prevision === 'NEGATIVO' && result.probabilidad > 0.65) {
    return 'MANUAL_REVIEW'; // Queue for review
  } else {
    return 'APPROVED'; // Safe to publish
  }
}
```

## Batch Analysis Insights

When analyzing bulk data, aggregate results to identify trends:

```javascript
function analyzeResultsTrends(resultados) {
  const stats = {
    totalPositive: 0,
    totalNegative: 0,
    avgConfidence: 0,
    highPriorityIssues: 0
  };
  
  resultados.forEach(result => {
    if (result.prevision === 'POSITIVO') {
      stats.totalPositive++;
    } else {
      stats.totalNegative++;
      if (result.probabilidad >= 0.80) {
        stats.highPriorityIssues++;
      }
    }
    stats.avgConfidence += result.probabilidad;
  });
  
  stats.avgConfidence /= resultados.length;
  
  return {
    ...stats,
    sentimentRatio: stats.totalPositive / resultados.length,
    needsAttention: stats.highPriorityIssues
  };
}
```

## Best Practices for Interpretation

1. **Don't rely solely on prevision** — Always consider the confidence score
2. **Set confidence thresholds** — Define minimum confidence levels for automated actions
3. **Review edge cases** — Manually check results with confidence between 50-60%
4. **Consider context** — Short texts may have lower accuracy than detailed feedback
5. **Track trends** — Monitor sentiment over time rather than individual results
6. **Validate regularly** — Periodically review predictions against human judgment
