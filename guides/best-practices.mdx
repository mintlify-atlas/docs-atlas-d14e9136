## Overview

This guide provides recommendations for using SentimentAPI effectively, securely, and efficiently. Follow these practices to maximize accuracy, performance, and security.

## When to Use Single vs Bulk Analysis

Choose the right analysis method based on your use case:

### Single Analysis

**Use when:**
- Analyzing individual user comments in real-time
- Integrating sentiment checks into live chat or support systems
- Testing the API or validating specific text samples
- Processing low-volume data (< 10 items)

**Implementation:**
```typescript
// Single comment analysis (app/sentiment/page.tsx:54-90)
const dataToAnalyze = [
  { texto: comment } 
];

const response = await fetch(`${process.env.NEXT_PUBLIC_API_URL}/sentiment`, {
  method: "POST",
  headers: { 
    "Content-Type": "application/json",
    "Authorization": `Bearer ${token}`
  },
  body: JSON.stringify(dataToAnalyze),
});
```

**Advantages:**
- Immediate results
- Lower memory overhead
- Real-time user feedback

<Tip>
Single analysis is ideal for interactive applications where users expect instant responses.
</Tip>

### Bulk Analysis (CSV Upload)

**Use when:**
- Processing historical data or batch imports
- Analyzing survey responses or feedback forms
- Periodic reporting (daily, weekly, monthly reviews)
- Processing high-volume data (10+ items)

**Implementation:**
```typescript
// Bulk CSV upload (app/sentiment/upload-csv/page.tsx:122-194)
const formData = new FormData();
formData.append('file', selectedFile);

const response = await fetch(`${process.env.NEXT_PUBLIC_API_URL}/sentiment/upload-csv`, {
  method: "POST",
  headers: { "Authorization": `Bearer ${token}` },
  body: formData,
});
```

**Advantages:**
- Efficient for large datasets
- Single request for multiple analyses
- Export and archive results easily
- Statistical insights from aggregated data

<Warning>
Bulk uploads are limited to 3MB file size. For larger datasets, split into multiple files.
</Warning>

### Comparison Table

| Feature | Single Analysis | Bulk Analysis |
|---------|----------------|---------------|
| **Max Items per Request** | 1 (though API accepts array) | Limited by 3MB file size |
| **Response Time** | Instant (< 1s) | Varies with file size |
| **Best for** | Real-time, interactive use | Batch processing, reports |
| **Export Options** | Manual copy | CSV download |
| **UI Location** | `/sentiment` | `/sentiment/upload-csv` |

## Performance Optimization

### 1. Minimize Token Usage

Authentication tokens are stored in cookies with a 2-hour expiration:

```typescript
// Session token configuration (app/actions/auth.ts:25-31)
cookieStore.set("session_token", data.token, {
  httpOnly: true,
  secure: process.env.NODE_ENV === "production",
  sameSite: "lax",
  maxAge: 60 * 60 * 2, // 2 hours
  path: "/",
});
```

<Tip>
**Best practice:** Implement token refresh logic before the 2-hour expiration to avoid session interruptions.
</Tip>

### 2. Optimize File Uploads

**Client-side validation** prevents unnecessary network requests:

```typescript
// Validate before upload (app/sentiment/upload-csv/page.tsx:138-143)
const MAX_SIZE_MB = 3 * 1024 * 1024;
if (selectedFile.size > MAX_SIZE_MB) {
  setMessage(`El archivo es demasiado pesado (${(selectedFile.size / (1024 * 1024)).toFixed(2)} MB). Por favor, sube uno menor a 3MB.`);
  setUploadStatus('error');
  return;
}
```

**Recommendations:**
- Validate file size before API call
- Check file format (.csv only)
- Display upload progress for better UX
- Implement retry logic for network failures

### 3. Handle Large Datasets Efficiently

For datasets exceeding 3MB:

```javascript
function splitCSVFile(lines, maxSizeBytes = 3 * 1024 * 1024) {
  const chunks = [];
  let currentChunk = ['texto']; // Header
  let currentSize = 0;
  
  for (let i = 1; i < lines.length; i++) {
    const lineSize = new Blob([lines[i]]).size;
    
    if (currentSize + lineSize > maxSizeBytes && currentChunk.length > 1) {
      chunks.push(currentChunk.join('\n'));
      currentChunk = ['texto'];
      currentSize = 0;
    }
    
    currentChunk.push(lines[i]);
    currentSize += lineSize;
  }
  
  if (currentChunk.length > 1) {
    chunks.push(currentChunk.join('\n'));
  }
  
  return chunks;
}
```

## Error Handling Patterns

Implement robust error handling based on actual patterns from the source code:

### 1. Network and Server Errors

```typescript
// Comprehensive error handling (app/sentiment/upload-csv/page.tsx:160-174)
try {
  const response = await fetch(url, options);
  
  if (!response.ok) {
    switch (response.status) {
      case 401:
        throw new Error("El archivo supera el límite de 3MB. Intenta con un archivo más pequeño.");
      case 413:
        throw new Error("El servidor indica que el archivo es demasiado grande.");
      case 403:
        throw new Error("No tienes permisos para realizar esta acción.");
      case 500:
        throw new Error("Error interno del servidor. Inténtalo más tarde.");
      default:
        throw new Error(`Ocurrió un problema inesperado (Error ${response.status})`);
    }
  }
} catch (err: any) {
  if (err.message === "Failed to fetch") {
    setMessage("No se pudo conectar con el servidor. Verifica tu conexión.");
  } else {
    setMessage(err.message);
  }
  setUploadStatus('error');
}
```

### 2. Authentication Errors


## Session Management Best Practices

### 1. Token Storage and Security

Tokens are stored as HTTP-only cookies for security:

```typescript
// Secure token storage (app/actions/auth.ts:24-31)
if (data.token) {
  const cookieStore = await cookies();
  cookieStore.set("session_token", data.token, {
    httpOnly: true,                              // Prevents XSS attacks
    secure: process.env.NODE_ENV === "production", // HTTPS only in production
    sameSite: "lax",                             // CSRF protection
    maxAge: 60 * 60 * 2,                         // 2-hour expiration
    path: "/",
  });
}
```

<Tip>
**Security benefits:**
- `httpOnly: true` prevents JavaScript access, protecting against XSS
- `secure: true` ensures HTTPS transmission in production
- `sameSite: "lax"` provides CSRF protection
- 2-hour expiration limits exposure window
</Tip>

### 2. Session Expiration Handling

Implement proactive session refresh:

```javascript
function setupSessionRefresh(expirationTime) {
  // Refresh 5 minutes before expiration
  const refreshTime = expirationTime - (5 * 60 * 1000);
  
  setTimeout(async () => {
    try {
      await refreshAuthToken();
    } catch (error) {
      // Redirect to login if refresh fails
      window.location.href = '/login';
    }
  }, refreshTime);
}
```

### 3. Graceful Session Expiry

Show clear messages when sessions expire:

```typescript
// Check session before critical operations
if (!token) {
  setMessage("Sesión expirada. Por favor, inicia sesión nuevamente.");
  setUploadStatus('error');
  return;
}
```

## Security Considerations

### 1. API Token Protection

<Warning>
**Never expose API tokens in:**
- Client-side code
- Git repositories
- Browser console logs
- URL parameters
- Local storage (use HTTP-only cookies instead)
</Warning>

### 2. Input Validation

Always validate user input before processing:

```typescript
// Validate file selection
if (!selectedFile) {
  setMessage("Por favor, selecciona un archivo primero.");

// Validate file type
```

## Rate Limiting and Usage Guidelines

### Recommended Usage Patterns

1. **Implement request throttling** for bulk operations
2. **Cache results** when analyzing the same content multiple times
3. **Batch requests** instead of making individual calls
4. **Monitor API response times** and adjust accordingly

### Example: Request Throttling

```javascript
class APIThrottler {
  constructor(maxRequestsPerMinute = 60) {
    this.maxRequests = maxRequestsPerMinute;
    this.requests = [];
  }
  
  async throttle() {
    const now = Date.now();
    this.requests = this.requests.filter(time => now - time < 60000);
    
    if (this.requests.length >= this.maxRequests) {
      const waitTime = 60000 - (now - this.requests[0]);
      await new Promise(resolve => setTimeout(resolve, waitTime));
    }
    
    this.requests.push(now);
  }
  
  async request(url, options) {
    await this.throttle();
    return fetch(url, options);
  }
}

const throttler = new APIThrottler(60);
```

## Data Privacy and Compliance

### 1. User Data Handling

- **Minimize data collection**: Only analyze necessary text
- **Clear data retention policies**: Define how long results are stored
- **Anonymize sensitive information**: Remove PII before analysis
- **Secure data transmission**: Always use HTTPS

### 2. GDPR Compliance

If processing EU user data:

- Obtain explicit consent before analysis
- Provide data export capabilities (CSV export feature)
- Allow users to delete their data
- Document data processing activities

### 3. Export and Audit

Use the CSV export feature for compliance and auditing:

