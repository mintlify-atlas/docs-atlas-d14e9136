## POST /sentiment/upload-csv

Upload a CSV file containing multiple text samples for batch sentiment analysis.

### Endpoint

```http
POST /sentiment/upload-csv
```

### Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token obtained from the `/login` endpoint
</ParamField>

### Request Headers

<ParamField header="Content-Type" type="string" required>
  Must be set to `multipart/form-data`
</ParamField>

<ParamField header="Authorization" type="string" required>
  Format: `Bearer YOUR_TOKEN`
</ParamField>

### Request Body

The request must include a file upload using multipart form data.

<ParamField body="file" type="file" required>
  CSV file containing text samples to analyze
  - **Max file size:** 3MB
  - **Format:** .csv
  - **Encoding:** UTF-8 recommended
</ParamField>

### File Format Requirements

- **Extension:** `.csv` only
- **Size limit:** 3MB (3,145,728 bytes)
- **Encoding:** UTF-8 for proper character support

### Request Examples

<CodeGroup>
```javascript JavaScript/FormData
const token = 'YOUR_TOKEN_HERE';
const fileInput = document.querySelector('#file-input');
const selectedFile = fileInput.files[0];

// Validate file size (3MB limit)

const results = await response.json();
```


```python Python
import requests
import os

token = os.getenv('AUTH_TOKEN')
url = f"{os.getenv('API_URL')}/sentiment/upload-csv"

headers = {
    'Authorization': f'Bearer {token}'
}

with open('comments.csv', 'rb') as file:
    files = {'file': file}
    response = requests.post(url, headers=headers, files=files)
    
if response.status_code == 200:
    results = response.json()
    print(f"Successfully analyzed {len(results)} entries")
else:
    print(f"Error: {response.status_code}")
```

### Client-Side Validation Example

```javascript
function validateFile(file) {
  const MAX_SIZE_MB = 3 * 1024 * 1024;
  
  if (!file) {
    return { valid: false, error: 'Por favor, selecciona un archivo primero.' };
  }
  
  if (file.size > MAX_SIZE_MB) {
    return { 
      valid: false, 
      error: `El archivo es demasiado pesado (${(file.size / (1024 * 1024)).toFixed(2)} MB). Por favor, sube uno menor a 3MB.`
    };
  }
  
  if (!file.name.endsWith('.csv')) {
    return { valid: false, error: 'Solo se aceptan archivos .csv' };
  }
  
  return { valid: true };
}
```

### Response

The API returns an array of sentiment analysis results for all texts in the CSV file.

<ResponseField name="texto" type="string">
  The original text from the CSV file
</ResponseField>

<ResponseField name="prevision" type="string">
  Sentiment prediction: `POSITIVO` or `NEGATIVO`
</ResponseField>

<ResponseField name="probabilidad" type="number">
  Confidence score between 0 and 1 (e.g., 0.92 = 92% confidence)
</ResponseField>

## Error Responses

<Expandable title="401 Unauthorized - File Too Large">
  **Response:**
  ```json
  {
    "error": "El archivo supera el límite de 3MB. Intenta con un archivo más pequeño."
  }
  ```
  
  **Status Code:** 401
  
  **Cause:** File size exceeds 3MB limit.
  
  **Solution:** Reduce file size or split into multiple smaller files.
</Expandable>

<Expandable title="413 Payload Too Large">
  **Response:**
  ```json
  {
    "error": "El servidor indica que el archivo es demasiado grande."
  }
  ```
  
  **Status Code:** 413
  
  **Cause:** Server-level file size restriction.
  
  **Solution:** Ensure file is under 3MB.
</Expandable>

<Expandable title="403 Forbidden">
  **Response:**
  ```json
  {
    "error": "No tienes permisos para realizar esta acción."
  }
  ```
  
  **Status Code:** 403
  
  **Cause:** Insufficient permissions or invalid token.
  
  **Solution:** Verify token validity and user permissions.
</Expandable>

<Expandable title="500 Internal Server Error">
  **Response:**
  ```json
  {
    "error": "Error interno del servidor. Inténtalo más tarde."
  }
  ```
  
  **Status Code:** 500
  
  **Cause:** Server processing error.
  
  **Solution:** Retry later or contact support.
</Expandable>

<Expandable title="Connection Error">
  **Response:**
  ```json
  {
    "error": "No se pudo conectar con el servidor. Verifica tu conexión."
  }
  ```
  
  **Cause:** Network connectivity issues or server unavailable.
  
  **Solution:** Check internet connection and server status.
</Expandable>

## Status Code Reference

| Status Code | Message | Cause |
|-------------|---------|-------|
| 200 | Success | File processed successfully |
| 401 | File size error | File exceeds 3MB limit |
| 403 | Forbidden | Insufficient permissions |
| 413 | Payload too large | Server-side size limit |
| 500 | Server error | Internal processing error |

## Complete Implementation Example

```javascript React Component
import { useState } from 'react';

function CSVUploader({ token }) {
  const [selectedFile, setSelectedFile] = useState(null);
  const [uploading, setUploading] = useState(false);
  const [results, setResults] = useState([]);
  const [error, setError] = useState('');

  const handleFileChange = (event) => {
    const file = event.target.files[0];
    if (file) {
      // Validate file size
      const MAX_SIZE_MB = 3 * 1024 * 1024;
      if (file.size > MAX_SIZE_MB) {
        setError(`File too large: ${(file.size / (1024 * 1024)).toFixed(2)} MB`);
        return;
      }
      setSelectedFile(file);
      setError('');
    }
  };

  const handleUpload = async (e) => {
    e.preventDefault();
    
    if (!selectedFile) {
      setError('Please select a file first');
      return;
    }

    setUploading(true);
    const formData = new FormData();
    formData.append('file', selectedFile);

    try {
      const response = await fetch(
        `${process.env.NEXT_PUBLIC_API_URL}/sentiment/upload-csv`,
        {
          method: 'POST',
          headers: { 'Authorization': `Bearer ${token}` },
          body: formData
        }
      );

      if (!response.ok) {
        switch (response.status) {
          case 401:
            throw new Error('File exceeds 3MB limit');
          case 413:
            throw new Error('File too large for server');
          case 403:
            throw new Error('Permission denied');
          case 500:
            throw new Error('Server error. Try again later');
          default:
            throw new Error(`Unexpected error: ${response.status}`);
        }
      }

      const data = await response.json();
      setResults(data);
      setSelectedFile(null);
      setError('');
    } catch (err) {
      setError(err.message);
    } finally {
      setUploading(false);
    }
  };

  return (
    <div>
      <form onSubmit={handleUpload}>
        <input 
          type="file" 
          accept=".csv" 
          onChange={handleFileChange}
        />
        <button type="submit" disabled={!selectedFile || uploading}>
          {uploading ? 'Processing...' : 'Upload & Analyze'}
        </button>
      </form>
      
      {error && <div className="error">{error}</div>}
      
      {results.length > 0 && (
        <div>
          <h3>Results: {results.length} comments analyzed</h3>
          {results.map((item, idx) => (
            <div key={idx}>
              <p>{item.texto}</p>
              <p>{item.prevision} ({(item.probabilidad * 100).toFixed(1)}%)</p>
            </div>
          ))}
        </div>
      )}
    </div>
  );
}
```

## Best Practices

1. **Validate file size client-side** before upload to save bandwidth
2. **Show upload progress** for better user experience
3. **Handle all error cases** with user-friendly messages
4. **Clear file input** after successful upload
5. **Provide CSV format examples** to users
6. **Test with various file sizes** near the 3MB limit

<Tip>
  For optimal performance, limit CSV files to 1,000-5,000 rows. Larger datasets should be split into multiple files.
</Tip>

<Warning>
  The file size limit is strictly enforced at 3MB (3,145,728 bytes). Files even slightly over this limit will be rejected.
</Warning>
